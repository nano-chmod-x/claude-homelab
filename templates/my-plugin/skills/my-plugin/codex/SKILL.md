# My Plugin Skill

Target path relative to plugin root: `skills/my-plugin/SKILL.md`

Use this as the Codex-facing version of the plugin skill.

Recommended sections:
- Purpose
- Trigger phrases
- Preferred tool path
- Fallback path
- Action reference
- Destructive-operation rules
- Examples

Available fields and concepts to document in the skill body:
- tool name
- action
- subaction
- pagination fields
- sort/order fields
- filter fields
- required ids or names
- confirmation requirements for destructive actions

Keep the action list aligned with the Claude-facing version and document the same destructive-operation rules.
